What this demo shows

  • Two direct FlexRenderer instances can attach to one shared WebGL context key.
  • Each shared renderer should still have its own presentation canvas.
  • A private renderer should still use one canvas for WebGL and presentation.
  • Shared clear() should clear only the selected renderer-local presentation canvas.
  • A busy shared context should skip or throw according to policy.

Expected after Create

  1. Shared refCount should be 2.
  2. Renderer A and B should share the same WebGL canvas.
  3. Renderer A and B should not share the same presentation canvas.
  4. The private renderer should have presentation canvas equal to WebGL canvas.

This is a direct renderer validation demo for the current implementation phase. It does not use FlexDrawer sharedContextKey yet.
