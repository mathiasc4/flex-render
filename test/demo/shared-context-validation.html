What this demo shows

  • Two real OpenSeadragon viewers use FlexDrawer with the same sharedContextKey.
  • FlexDrawer passes the key into FlexRenderer through drawerOptions.
  • The two shared drawers use one WebGL backing canvas but separate presentation canvases.
  • A third viewer omits sharedContextKey and remains private.
  • Shared renderer output is presented through each renderer's local presentation canvas.
  • Shared clear() clears only the selected renderer-local presentation output.
  • Shared viewer A and B are stacked vertically and deliberately use different CSS heights.
  • Resize presets verify that each shared renderer keeps an independent presentation canvas size.

Expected after Create viewers

  1. All three viewer panels should visibly render their labeled tile pattern.
  2. Shared context status should contain one entry for the configured shared key.
  3. The shared entry refCount should be exactly 2 because navigators are disabled.
  4. Shared viewer A and B should share the same WebGL canvas.
  5. Shared viewer A and B should use different presentation canvases.
  6. The private viewer should report private-context mode.
  7. The WebGL context usage panel should report 3 renderers using 2 WebGL contexts.
  8. Every item in the Checks panel should pass after all viewers finish loading.
  9. Shared viewer A and B should have different presentation canvas sizes that match their own render dimensions.

Expected after Run clear check

  1. Only Shared viewer A should become visually blank.
  2. Shared viewer B should remain visible.
  3. The private viewer should remain visible.
  4. The latest action should report passed: true.
  5. The alpha readout should show Shared A dropping to 0 while Shared B remains above 0.

Expected after Run resize check

  1. The demo should switch the shared viewer size preset.
  2. Shared viewer A and B should both redraw visibly.
  3. Shared viewer A and B should still use the same WebGL context.
  4. Shared viewer A and B should keep different presentation canvas sizes.
  5. Each renderer presentation canvas should match only its own render dimensions.
  6. The latest action should report passed: true.

Expected after Reload sources

  1. Shared viewer A should render again after being cleared.
  2. Shared viewer B and the private viewer should continue rendering normally.
  3. All Checks panel items should pass again after the viewers finish loading.

Shared-context presentation currently uses readPixels only. The previous default-framebuffer blit transfer path was removed because it is not reliable across browser/context configurations.
