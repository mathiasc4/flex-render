What this demo shows

  • The custom demo TileSource returns normal canvas tiles for most coordinates.
  • Selected coordinates return invalid data intentionally.
  • FlexDrawer should convert those conversion failures into first-pass diagnostics.
  • FlexRenderer should render those diagnostics as a warning glyph and border inside the affected tile.

Checks

  1. With diagnostics enabled, invalid tiles should show the warning marker.
  2. With diagnostics disabled, invalid tiles should not render warning markers.
  3. Changing the invalid pattern and reloading should produce deterministic affected tiles.
  4. The shader layer controls should still behave normally.

The invalid data is deterministic and demo-local. It does not depend on network errors or CORS behavior.
